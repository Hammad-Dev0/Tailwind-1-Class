# Tailwind CSS States & Responsive Design

This project demonstrates various **Tailwind CSS states** and **responsive design techniques** using HTML and Tailwind CDN.

## ğŸ“Œ Features

### **1. Tailwind CSS States**
- **Hover State:** Change button background on hover.
- **Focus State:** Input highlights on focus.
- **Active State:** Buttons change when active.
- **Visited State:** Links change color after visiting.
- **Disabled State:** Buttons visually disabled.
- **Checked State:** Checkbox styling when checked.
- **Focus-visible State:** Outline ring on focus-visible.
- **Focus-within State:** Container border changes when child input is focused.
- **Read-only State:** Styling for read-only inputs.
- **Required State:** Input field validation.
- **Invalid State:** Input validation styling.
- **Group Hover State:** Text changes when parent is hovered.
- **Placeholder State:** Input placeholder styling.

### **2. Responsive Design**
- Background color changes for `sm`, `md`, and `lg` breakpoints.
- Flexbox layout changes from column to row on medium screens.

### **3. Custom Styling**
- Hex and LCH colors for backgrounds.
- Tailwind preflight demonstration.
